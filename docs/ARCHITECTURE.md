# Архитектура MVP

## Общая модель
Однопроцессное tray‑приложение с event‑driven моделью.

## Основные модули
- `app` — управление состояниями и маршрутизация событий.
- `hotkey` — глобальная горячая клавиша.
- `audio` — запись с микрофона и буфер в памяти.
- `stt` — интеграция с Groq API.
- `inject` — вставка текста в активное окно.
- `tray` — иконка и меню в трее.
- `settings` — окно настроек.
- `config` — сохранение настроек.
- `secret` — шифрование API‑ключа (DPAPI).
- `logger` — логирование с ротацией.
- `autostart` — автозапуск через HKCU Run.

## Состояния
- `Idle`
- `Recording`
- `Transcribing`
- `Injecting`
- `Checking`
- `Error`

## Поток работы
1. Hotkey → старт записи.
2. Stop/timeout → завершение записи.
3. Отправка в Groq API.
4. Вставка текста.
5. Возврат в `Idle`.
